What types of documents will users upload?

Select all that apply — this shapes which OCR engine and pipeline stages we need

A

Handwritten Forms

Paper forms filled out by hand — insurance, medical, government, intake sheets. Needs HTR (handwritten text recognition).

B

Scanned Typed / Printed Documents

Typed documents scanned to PDF or image — invoices, contracts, reports. Standard OCR handles these well.

C

Native Digital PDFs

PDFs with selectable text already embedded (no OCR needed) — exported from Word, Excel, accounting software.

D

Mixed / Hybrid

A combination of the above — e.g., a typed form with handwritten fill-ins, or batches that include any of the above types.
