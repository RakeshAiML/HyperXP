Simplified POC Pipeline — 4 Pages

Scoped to the BPR Raw Material section (Pages 1–4)

1

Upload PDF

User uploads the 4-page BPR PDF via a simple web UI. Backend converts all 4 pages to PNG images using PyMuPDF at 300 DPI.

↓
2

GPT-4o Vision Extraction

All 4 images sent to GPT-4o Vision in a single API call with two schema prompts:

• Header schema (Page 1 only) — Product, Batch No, dates, sign-off fields
• Raw Material schema (Pages 1–4) — all 35 rows: material name, UOM, standard qty, charged qty, AR No., equipment ID, remarks, performed/checked sign+date

↓
3

Validate & Clean

Simple checks: required fields present, quantities are numeric, dates parseable. Dashes (—) treated as null. Stars (* /**) on material names captured as a "flag" column. Any uncertain field highlighted yellow in Excel.

↓
4

Generate Excel & Download

JSON → openpyxl workbook with 2 sheets:
• Sheet 1: BPR Header — all top-level fields
• Sheet 2: Raw Materials — one row per material (35 rows), all columns

Download button appears in UI when ready.

POC scope

4 pages · 2 Excel sheets · ~35 raw material rows · 1 header block · ~3 API calls total · No database — just upload → Excel download
