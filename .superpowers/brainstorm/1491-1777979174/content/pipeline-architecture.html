BPR Extraction Pipeline — Architecture

End-to-end flow from PDF upload to structured Excel output

1

Upload & Convert

User uploads PDF via web UI → Backend converts each page to a high-res PNG image (300 DPI) using pdf2image / PyMuPDF. Handles multi-page BPRs (87 pages).

↓
2

Page Classification

GPT-4o Vision identifies the section type of each page — Header, Raw Material Table, Process Steps, In-Process Checks, Sign-off page — so we know which extraction schema to apply.

↓
3

GPT-4o Schema Extraction

Each page image + its schema prompt is sent to GPT-4o Vision. Returns structured JSON with all field values — including handwritten quantities, lot numbers, dates, and signature presence (✓/✗). Runs pages in parallel.

↓
4

Validation & Merge

Rule-based checks: required fields present, numeric quantities reasonable, dates parseable. Multi-page tables (Raw Material rows 1–35 across pages 1–4) are stitched together. Low-confidence fields flagged for review.

↓
5

Excel Generation

Structured JSON → openpyxl Excel workbook with separate sheets: BPR Header, Raw Materials, Process Steps, etc. Flagged cells highlighted in yellow. Download via web UI.
