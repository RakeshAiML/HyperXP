What gets built

A web app where a user uploads the 4-page BPR PDF and gets back a structured Excel file with 2 sheets — no database, no login, no fluff.

Excel Output — 2 Sheets

Sheet 1 · BPR Header

22 columns · 1 row per BPR
Product, Batch No, dates, durations, all 4 sign-off fields (Yes/No + date each), QA stamp, Form No

Sheet 2 · Raw Materials

14 columns · 35 rows (all materials)
Name, UOM, Std Qty, Charged Qty, AR No, Eq ID, Remarks, Performed+Checked (sign+date), Flag, Confidence

Low-confidence cells highlighted yellow · Dashes extracted as empty · * and ** captured in Flag column

GPT-4o Prompting

All 4 page images sent in one API call with two JSON schemas embedded in the prompt — one for the header block, one for the raw materials table. Model instructed to: use null for dashes, flag ambiguous handwriting as Low confidence, normalize all dates to DD/MM/YYYY.
